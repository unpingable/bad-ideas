# bad-ideas

Quarantine for designs that correctly diagnose a real pressure but smuggle in an implementation project larger than the problem.

These are genuine bad ideas — recurrent substrate temptations that keep failing examination — not ha-ha-this-worked-out war stories. The point of filing them publicly is that public quarantine is harder to escape from than private quarantine.

## Two species, separate shelves

- **`entries/`** — temptations with receipts. Things repeatedly tried (or attempted to sneak past review) that keep failing on examination. Records the seductive shape, real pressure named, why building is wrong, the useful isotope to extract, the falsification condition, and the keeper line.
- **`patterns/`** *(not yet populated)* — recurring architectural sirens that catch many people, not just one operator. Lower bar for novelty; higher bar for entry (must demonstrate recurrence beyond a single operator).

## Rule

> An entry records a temptation with receipts. A pattern records a recurring architectural hallucination. **Do not confuse diagnosis with permission.**

Temptations file themselves. Patterns require recurrence.

## Relation to `rhizome`

Sibling discipline, separate scope. [`rhizome`](https://github.com/unpingable/rhizome) is the *positive* evidence/design-memo locker — static artifacts, thresholds, negative defaults, bounded design doctrine for federation. Its README is uncompromising: *"This place is not a place of honor. What is here is not a project."*

`bad-ideas` is the *negative-pressure companion* — designs that correctly identify a pressure but are quarantined because building them would likely widen scope, create substrate debt, or convert diagnosis into platform ambition.

`bad-ideas` may cite `rhizome` for precedent on static-containment style. It is **not** a submodule, extension, or implementation lane of `rhizome`.

> **Rhizome proves the containment style; it does not authorize the contained idea.**

Citation is not authorization. Authority laundering wearing a markdown hat is exactly the failure mode this repo exists to catch.
